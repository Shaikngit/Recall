# Recall Workspace Instructions

## Private Endpoints for PaaS Connectivity — CRITICAL RULES

**DO NOT ignore these rules.** They are required for secure, working connectivity.

### Storage Account & Blob Access

- **Storage account**: `shaiknrecall` in resource group `rg-mykb-shaikn`
- **Private Endpoint**: `pe-shaiknrecall-blob` connected to private DNS zone `privatelink.blob.core.windows.net`
- **App Service**: `app-mykbshaikn-th6h7z` has VNet integration via subnet `appsvc-integration-subnet`
- **VNet**: `vm-mykb-ghrunner-01VNET`

### Essential Configuration

When deploying or modifying infrastructure:

1. **VNet Integration**: App Service MUST have VNet integration to access storage via private endpoint
2. **App Settings** (required for DNS resolution):
   - `WEBSITE_VNET_ROUTE_ALL=1`
   - `WEBSITE_DNS_SERVER=168.63.129.16`
3. **Storage publicNetworkAccess**: Must be **Disabled** at all times (except during manual blob uploads)
4. **Private Endpoint**: MUST exist with private DNS zone integration

### Never Remove

- VNet integration on App Service
- Private endpoint `pe-shaiknrecall-blob`
- Private DNS zone `privatelink.blob.core.windows.net`
- DNS VNet link
- App settings `WEBSITE_VNET_ROUTE_ALL` and `WEBSITE_DNS_SERVER`

If any Bicep/Terraform/CLI deployment would remove these, **abort and fix the template first**.

### Blob Upload Workflow

Public network access is normally Disabled. To upload blobs manually:

1. **Enable temporarily**: 
   ```powershell
   az storage account update -n shaiknrecall -g rg-mykb-shaikn --public-network-access Enabled -o none
   ```
2. **Upload**:
   ```powershell
   az storage blob upload --account-name shaiknrecall --container-name mykb-content --name "<blob-path>" --file "<local-file>" --overwrite --auth-mode login -o none
   ```
3. **Disable immediately**:
   ```powershell
   az storage account update -n shaiknrecall -g rg-mykb-shaikn --public-network-access Disabled -o none
   ```

- Always use `--auth-mode login` (managed identity / CLI auth), never storage keys
- Never leave public access enabled — disable right after upload

### Managed Identity

- App Service `app-mykbshaikn-th6h7z` has system-assigned identity `6dc7a97d-3869-4a7d-ba74-29bc3c5b3a9f`
- Identity has **Storage Blob Data Contributor** role on `shaiknrecall`
- Storage access is blocked if VNet integration or private endpoint is missing
